# Documentation

- `project.org` — living status, priorities, and dated progress notes.
- `renderer-design.md` — renderer and platform architecture.
- `surface-atlas.md` — current surface domains, typed sections, and renderer
  presentation bridge.
- `terrain-expressions.md` — terrain program and evaluator design.
- `trails.md` — leisure-circuit planning, grading, materials, and consumers.
- `units.md` — scientific quantities, unit conventions, and simulation scale.
- `working-practices.md` — development and verification practices.

The Atelier charters (`atelier-earth.md` and `atelier-tree.md`) describe the
architectural line that the current engine may adopt organ by organ. They are
proposals and proofs, not a claim that Moppe has already been rewritten.

Long-range design writing lives in `../ideas/`.  Those documents explain
possible directions, but do not claim that their proposed systems exist.
